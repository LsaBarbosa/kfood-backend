# Store Terms Acceptance Data Dictionary

## Table

- name: `store_terms_acceptance`
- purpose: persist the legal terms acceptance history of a merchant store for compliance and audit traceability

## Source of truth

- JPA entity: `src/main/java/com/kfood/merchant/infra/persistence/StoreTermsAcceptance.java`
- baseline migration: `src/main/resources/db/migration/V1__baseline.sql`
- forward migration for request IP: `src/main/resources/db/migration/V22__store_terms_acceptance_request_ip.sql`

## Columns

| Column | Type | Required | Semantics |
| --- | --- | --- | --- |
| `id` | `uuid` | yes | Unique identifier of the acceptance record. |
| `store_id` | `uuid` | yes | Store that owns the acceptance history entry. Foreign key to `store(id)`. |
| `accepted_by_user_id` | `uuid` | yes | Authenticated user who accepted the legal document. Foreign key to `identity_user(id)`. |
| `document_type` | `varchar(40)` | yes | Legal document type accepted by the store user, for example `TERMS_OF_USE`. |
| `document_version` | `varchar(40)` | yes | Accepted legal document version. |
| `accepted_at` | `timestamp` | yes | Acceptance timestamp generated by the backend at the moment of the operation. |
| `request_ip` | `varchar(45)` | yes | Client IP captured from the HTTP request metadata by the backend and persisted as part of the technical compliance trail. This field is internal and is not part of the public API request or response payload. |
| `created_at` | `timestamp` | yes | Database creation timestamp of the persistence record. |

## Notes

- `accepted_at` is server-side data. The client does not control this field.
- `request_ip` is derived and normalized by the backend from the HTTP request metadata before persistence.
- The public API contract for terms acceptance does not expose `request_ip` in request or response payloads.
